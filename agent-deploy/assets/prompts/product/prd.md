## 제약조건

- 요구사항은 검증 가능한 단위로 작성
- 추측이 필요하면 가정을 먼저 밝힌다
- 외부 자료 사용 시 출처 표기

## 출력

1. 문제 정의 (대상 / 문제 / 근거)
2. 목표 / 성공 지표
3. 요구사항 (R1, R2 ...)
4. Non-goals (이번에 안 하는 것)
5. 수용 기준 (관찰 가능)
6. 리스크 / 미결
