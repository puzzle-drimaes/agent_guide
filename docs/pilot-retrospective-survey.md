# Pilot 회고 설문 / 인터뷰 문항 (2주 종료 후)

2주 Pilot 종료 직후 바로 쓸 수 있는 피드백 수집 도구다.
[10-pilot.md](plans/codex/company-wide-agent-rollout/10-pilot.md)의 짧은 회고 질문을 설문·인터뷰로 구체화한 것이다.

## 사용법

```text
- Part A 설문: 전 참가자, 약 5분. Slack/Notion 폼으로 받는다. 5점 척도 + 짧은 서술.
- Part B 인터뷰: 희망자 또는 직무 대표 1~2명, 15~30분. 설문 답을 보고 심화한다.
- 익명/실명 선택 가능. 답변은 개인 평가가 아니라 운영 개선용으로만 쓴다.
- 척도: 1 = 전혀 아니다 … 3 = 보통 … 5 = 매우 그렇다.
```

---

## Part A. 설문 (전 참가자)

### A1. 기본 정보

```text
- 직무: 개발 / PM·기획 / 사업·영업·CS / 운영·governance
- 주로 쓴 AI 도구: Claude / Codex / Gemini
- OS: Windows / macOS / Linux
- 설치 여부: 정식 설치함 / 설치 없이 사용 / 시도했지만 실패
```

### A2. 설치 난이도

```text
(척도) 1. 설치 안내(SETUP_WIZARD / 따라하기)만 보고 혼자 진행할 수 있었다.
(척도) 2. dry-run(미리보기)으로 무엇이 설치될지 이해할 수 있었다.
(척도) 3. 막혔을 때 "설치가 안 될 때" 안내로 스스로 해결할 수 있었다.
(서술) 4. 설치 중 가장 막힌 지점은 무엇이었고, 어떻게 해결/포기했나?
(단답) 5. 설치에 걸린 대략 시간:
```

### A3. AI 도구별 사용성

```text
(척도) 1. 같은 회사 규칙인데도 내가 쓴 도구에서 잘 작동했다.
(척도) 2. 도구가 회사 표준 방식(profile/프롬프트)대로 응답했다.
(서술) 3. 도구별로 결과가 달랐던 사례가 있나? (어느 도구에서 무엇이 달랐는지)
(단답) 4. 가장 손에 맞았던 도구와 그 이유:
```

### A4. 공유 폴더 이해도

```text
(척도) 1. 공유 폴더(AI-Knowhow)에서 .md를 받아 쓸 수 있었다.
(척도) 2. 내 .md를 올리는 방법을 안다(원하면 올릴 수 있다).
(척도) 3. skills 폴더와 prompts 폴더의 구분을 안다.
(서술) 4. 공유 폴더에서 헷갈렸던 점은?
(단답) 5. 받아 쓰거나 올린 .md 개수(대략):
```

### A5. 보안 룰 이해도

```text
(척도) 1. 무엇을 입력하면 안 되는지(금지 데이터)를 안다.
(척도) 2. 민감정보를 placeholder(<고객명> 등)로 바꾸는 법을 안다.
(척도) 3. 외부 자료를 쓰면 출처를 남겨야 한다는 것을 안다.
(서술) 4. 보안 룰을 지키기 어렵거나 애매했던 상황은?
(예/아니오) 5. 금지 데이터를 입력할 뻔했거나 입력한 적이 있나? (있으면 상황을 적어주세요)
```

### A6. 전체 평가

```text
(척도) 1. 전사로 확대해도 좋다고 본다.
(서술) 2. 동료에게 가장 먼저 추천할 한 가지:
(서술) 3. 확대 전에 꼭 고쳐야 할 한 가지:
(단답) 4. 0~10점, 동료에게 추천하는 정도:
```

## Part B. 인터뷰 가이드 (심층, 희망자/대표)

설문 답을 먼저 보고, 아래에서 필요한 것만 골라 깊게 묻는다.

### B1. 설치 경험

```text
- 설치를 처음부터 끝까지 말로 재현해 주세요. 어디서 멈칫했나요?
- (비개발자) AI에게 SETUP_WIZARD를 주고 "시키는" 설치 방식이 직관적이었나요?
- 다시 한다면 어떤 안내가 더 있었으면 했나요?
```

### B2. 도구 / 워크플로

```text
- 이번에 실제로 끝낸 업무 1건을 구체적으로 말해 주세요. 어떤 프롬프트/스킬을 썼나요?
- 회사 표준 방식이 기존 본인 방식보다 나았나요, 불편했나요?
- 도구를 바꿔 써본 적이 있다면 차이는 무엇이었나요?
```

### B3. 공유 문화

```text
- 공유 폴더에서 받아 쓴 경험 / 올린 경험을 말해 주세요.
- 올리는 게 부담됐나요? 무엇이 부담을 줄이거나 늘렸나요?
- (핵심) "강제 아님, 공유하고 싶을 때만"이라는 방침이 잘 전달됐나요?
```

### B4. 보안

```text
- 민감정보 판단이 애매했던 실제 사례가 있었나요?
- 무엇이 있으면 더 안전하게 쓸 수 있을까요?
```

### B5. 확대

```text
- 본인 팀에 확대한다면 가장 큰 장벽은 무엇일까요?
- 직무별로 더 필요한 프롬프트/스킬은 무엇인가요?
```

## 집계 / 활용

```text
- 척도 문항은 평균과 분포만 본다. 개인 점수화·순위 매기기는 하지 않는다(운영 개선 목적).
- 서술/인터뷰는 주제별로 묶어 blocker / 개선 / 추천 으로 분류한다.
- 결과는 10-pilot.md "Pilot 종료 산출물"과 13-governance 분기 리뷰의 입력으로 쓴다.
- A5의 보안 위험 신호(금지 데이터 입력 등)는 집계와 별개로 governance 담당에게 바로 공유한다.
```
