# 09. 교육과 온보딩

## 목적

설치만 해놓고 사용하지 않는 상황을 막고, 전 직원이 같은 AI 업무 방식을 이해하게 한다.

## 교육 구조

### 1차 교육: 전 직원 60분

내용:

```text
- 왜 공용 계정을 쓰는가
- agent installer가 무엇을 설치하는가
- profile 선택 방법
- 보안/출처 표기 원칙
- 프롬프트 자산화 방법
- Drive 운영 메모 / 공유 폴더 / governance 기록 방식
```

실습:

```text
- 본인 profile 선택
- installer dry-run 실행
- 첫 업무 예시 1개 실행
- 결과를 회고 폼에 남기기
```

### 2차 교육: 직무별 45분

트랙:

```text
- 개발자
- PM/기획
- 디자인
- 운영/CS/세일즈
- 리더/매니저
```

직무별 실습:

```text
개발자: PR 노하우 작성
PM: 요구사항 문서 초안 작성
디자인: UX 리뷰 프롬프트 사용
운영/CS: 고객 응대 초안 작성
리더: 회의록/리스크 요약
```

### 3차 교육: 계정 담당자 30분

내용:

```text
- 주간 회고 작성
- 계정별 사용 사례 정리
- 실패 사례 공유 방식
- 신규 프롬프트 승인/정리
- 분기 리뷰 준비
```

## 교육 자료

```text
docs/install-guide.md
docs/profile-selection-guide.md
docs/non-developer-guide.md
docs/account-sharing-policy.md
docs/prompt-db-guide.md
docs/troubleshooting.md
```

## 교육 후 확인

```text
- 본인 profile을 말할 수 있는가?
- 설치 명령을 실행했는가?
- 첫 사용 예시를 수행했는가?
- 문제 보고와 노하우 공유 기록 위치를 아는가?
- 재사용 프롬프트를 어디에 등록하는지 아는가?
```

## 완료 기준

- 전 직원이 본인 profile을 알고 있다.
- 최소 80% 이상이 installer를 실행했다.
- 각 AI 계정 담당자가 지정되어 있다.
- 공유 폴더 위치와 .md 올리는/적용하는 방법을 안다 (회고 제출은 강제하지 않음).
