# 05. 공용 AI 계정 운영 정책

## 목적

20명 내외 구성원이 9개 AI 계정을 공유하면서도 혼란 없이 지식을 축적하도록 한다.

## 기본 원칙

```text
- 계정은 개인 소유가 아니라 팀 자산이다.
- 공용 계정의 목적은 비용 절감보다 지식 공유화다.
- 각 계정에는 초기 담당자가 있다.
- 공동 사용자는 사용 후 핵심 노하우를 남긴다.
- 좋은 프롬프트와 workflow는 Notion DB에 등록한다.
- 민감정보 입력은 금지한다.
- 계정별 사용 목적을 느슨하게 분리한다.
```

## 계정 역할 배정 예시

| 계정 | 주 용도 | 초기 담당 |
|---|---|---|
| AI-01 | 개발/코드리뷰 | 개발 리드 |
| AI-02 | 개발/테스트/디버깅 | 백엔드 |
| AI-03 | 기획/요구사항/정책 | PM |
| AI-04 | 디자인/UX/카피 | 디자인 |
| AI-05 | 운영/CS/FAQ | 운영 |
| AI-06 | 세일즈/제안서 | 세일즈 |
| AI-07 | 리서치/문서요약 | 공용 |
| AI-08 | 경영/보고/회의록 | 경영지원 |
| AI-09 | 실험/신규 툴 검증 | AI governance |

## 계정 사용 기록

최소 기록 항목:

```text
- 날짜
- 계정
- 사용자
- 목적
- 사용한 profile
- 사용한 AI 도구
- 좋은 결과 여부
- 재사용 가능한 프롬프트 링크
- 주의할 점
```

## Notion DB 필드

```text
Account ID
Primary Purpose
Initial Owner
Allowed Users
Weekly Summary
Best Prompt
Failure Case
Risk Note
Last Reviewed At
```

## 계정 담당자 책임

```text
- 주간 회고 1차 작성
- 공동 사용자 보강 요청
- 좋은 사례/실패 사례 수집
- 계정 사용 목적 변경 필요 시 #ai-governance에 제안
- 분기 리뷰 때 계정 재배분 의견 제출
```

## 금지 사항

```text
- 개인 비밀번호 저장
- 고객 개인정보 입력
- 계정별 공유 규칙 무시
- 좋은 프롬프트를 개인만 보관
- 출처 없는 외부 자료를 회사 산출물에 반영
```

## 완료 기준

- 9개 계정에 주 용도와 초기 담당자가 지정된다.
- 계정 사용 기록 템플릿이 있다.
- 계정 공유 정책이 전 직원에게 공지된다.
- 계정별 주간 회고 책임자가 명확하다.
