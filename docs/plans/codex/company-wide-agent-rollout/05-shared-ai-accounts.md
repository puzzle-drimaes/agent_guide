# 05. 공용 AI 계정 운영 정책

## 목적

20명 내외 구성원이 9개 AI 계정을 공유하면서도 혼란 없이 지식을 축적하도록 한다.

## 기본 원칙

```text
- 계정은 개인 소유가 아니라 팀 자산이다.
- 공용 계정의 목적은 비용 절감보다 지식 공유화다.
- 각 계정에는 초기 담당자가 있다.
- 공동 사용자는 공유하고 싶은 노하우가 있으면 가볍게 남긴다 (강제 아님).
- 좋은 프롬프트와 workflow는 원하면 Google Drive 공유 폴더와 GitHub prompts/skills branch에 등록한다.
- 민감정보 입력은 금지한다.
- 계정별 사용 목적을 느슨하게 분리한다.
```

## 전 계정 공통 금지 데이터 (D07 baseline)

아래는 **모든 계정에 공통 적용**되는 P0 입력 금지 범위다. 계정별 표의 "특히 주의" 칸은 이 baseline에 더해 해당 업무에서 사고 위험이 큰 항목을 강조한 것이다.

```text
- 고객 개인정보
- API key / token / password / private key
- 미공개 계약 조건
- 민감한 재무 정보
- 법무 검토 전 외부 공개 불가 자료
- 사내 계정 credential
→ 입력 전 placeholder로 치환, 판단 어려우면 담당자/보안 검토 후 사용
```

## 계정 운영/매핑표 (초안)

업무 유형별 배정 + 계정별 초기 담당자 1명 + 공동 사용자 N명(D06). 담당자/공동 사용자 실명은 Pilot 킥오프 시 확정한다.

| 계정 | 주 용도(사용 목적) | 초기 담당 | 공동 사용자 | 특히 주의 금지 데이터 | 지식 공유 태그(직무) |
|---|---|---|---|---|---|
| AI-01 | 개발/코드리뷰 | 개발 리드 | 개발팀 N명 | 소스 내 credential/key, 고객 데이터 샘플 | developer |
| AI-02 | 개발/테스트/디버깅 | 백엔드 | 개발팀 N명 | 운영 DB 덤프, 실데이터, token | developer |
| AI-03 | 기획/요구사항/정책 | PM | 기획/PM N명 | 미공개 로드맵, 미공개 계약 조건 | product |
| AI-04 | 디자인/UX/카피 | 디자인 | 디자인/기획 N명 | 미공개 브랜드/계약 자산 | product |
| AI-05 | 운영/CS/FAQ | 운영 | 운영팀 N명 | 고객 개인정보(최우선) | business |
| AI-06 | 세일즈/제안서 | 세일즈 | 세일즈 N명 | 미공개 계약/가격, 고객 명단 | business |
| AI-07 | 리서치/문서요약 | 공용 | 전사 | 출처 미확인 외부 자료, 라이선스 불명 자료 | 공용/research |
| AI-08 | 경영/보고/회의록 | 경영지원 | 경영/리드 N명 | 민감 재무, 법무 검토 전 자료 | governance/business |
| AI-09 | 실험/신규 툴 검증 | AI governance | 검증 참여자 | 신규 툴에 실데이터 투입 금지(검증은 더미로) | governance |

## 계정 사용 기록(선택 공유 시 권장)

모든 사용을 기록하지 않는다. 공유하고 싶은 사례가 있거나 계정 운영자가 주간 요약을 만들 때 아래 항목을 선택적으로 남긴다.

권장 기록 항목:

```text
- 날짜
- 계정
- 사용자
- 목적
- 사용한 profile
- 사용한 AI 도구
- 좋은 결과 여부
- 재사용 가능한 프롬프트 링크
- 주의할 점
```


## 계정 담당자 역할

계정 담당자는 감시자나 제출률 관리자 역할이 아니다. 공용 계정의 목적과 안전선을 유지하면서 좋은 사례가 자연스럽게 공유되도록 돕는다.

```text
- 공유할 만한 사례가 보이면 한 줄 정리를 제안한다 (강제 아님).
- 좋은 사례/실패 사례를 모아 가끔 요약한다.
- 공동 사용자가 보안/출처 규칙을 헷갈리면 가이드한다.
- 계정 사용 목적 변경 필요 시 Drive 운영 메모 또는 GitHub commit/issue 후보에 남긴다.
- 분기 리뷰 때 계정 재배분 의견을 제출한다.
```

## 지식 공유 루프

공용 계정의 목적은 비용 절감이 아니라 지식 공유화다(D06). 다만 이 루프는 **강제 절차가 아니라 가벼운 권장**이다.
모든 사용을 기록할 필요는 없고, **공유하고 싶다면** 부담 없이 남기면 된다.

```text
계정 사용
  → 공유하고 싶다면 한 줄로 남긴다 (매 사용마다 기록할 필요 없음)
  → agent가 쓸 만한 스킬/프롬프트 `.md`를 정제한다
  → Google Drive에 올리고 GitHub prompts/skills branch에 남긴다
  → 다른 사원이 Drive 또는 후보 branch에서 선택해 자기 프로젝트에 적용한다 (AI-Knowhow/)
  → 후보 branch 자료는 "검증 전 공유본"으로 표시하고 사용자 컨펌을 받는다
  → 반복해서 쓸 만한 프롬프트만 main 수동 병합 또는 승격 후보로 표시한다 (표의 "지식 공유 태그" = 대상 직무)
  → 충분히 검증되면 main 또는 company-* skill로 승격한다
```

지키는 선(나머지는 모두 best-effort):

```text
- 보안: 금지 데이터 미입력은 예외 없이 지킨다.
- 승격: company-* 승격만 검증/승인 게이트를 거친다.
- 그 외(매 사용 기록·회고·등록)는 강제하지 않고 권장한다.
```

루프 책임(권장):

```text
- 마중물:     계정 초기 담당자가 공유 분위기를 이끈다 (감시자가 아니다).
- 직무 태깅:  GitHub 후보 branch 등록 시 표의 "지식 공유 태그" 직무로 분류.
- 검증/승격:  AI governance + PMO (분기 리뷰).
```

연계 문서:

```text
- 회고/후보 branch/KPI:  06-knowledge-kpi-system.md
- prompt 후보 등록·승격:  15-prompt-db-operations.md
- 분기 리뷰/RACI:       13-governance-and-roadmap.md
```

## 금지 사항

```text
- 개인 비밀번호 저장
- 고객 개인정보 입력
- 계정별 보안/출처 규칙 무시
- 검증되지 않은 프롬프트를 company-* 표준 asset처럼 배포
- 출처 없는 외부 자료를 회사 산출물에 반영
```

## 완료 기준

- 9개 계정에 주 용도와 초기 담당자가 지정된다.
- 선택 공유용 계정 사용 기록 템플릿이 있다.
- 계정 공유 정책이 전 직원에게 공지된다.
- 계정별 마중물 담당자와 보안/출처 에스컬레이션 경로가 명확하다.
