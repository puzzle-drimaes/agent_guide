# 10. Pilot 운영

## 목적

전사 배포 전에 작은 그룹에서 설치, profile, 공용 계정 운영, 지식 공유 룰의 문제를 발견한다.

## 기간

```text
2주
```

## Pilot 대상

권장 구성:

```text
- 개발자 2명
- PM/기획 1명
- 비개발 직군 1명
- AI governance 담당 1명
```

## Pilot 전 준비

```text
- minimal/developer/product/business profile 준비
- Codex/Claude target 동작 확인
- 설치 가이드 초안 작성
- #ai-help, #ai-knowhow, #ai-governance 채널 준비
- Prompt DB 템플릿 준비
- 회고 폼 준비
```

## 1주차 검증

설치 검증:

```text
- Windows/macOS/Linux 중 가능한 OS에서 설치
- Codex/Claude/Gemini 중 가능한 target에서 설치
- dry-run과 apply 비교
- doctor 결과 확인
```

업무 적용:

```text
- 개발자: PR 노하우 1개 작성
- PM: 요구사항 초안 1개 작성
- 비개발자: 회의록/고객응대/제안서 중 1개 작성
```

## 2주차 검증

운영 검증:

```text
- 주간 회고 작성
- Prompt DB 등록
- 계정 담당자 회고
- 실패 사례 1개 이상 수집
- profile/rule 수정 제안 수집
```

## Pilot KPI

```text
- 설치 성공률
- 설치 평균 소요 시간
- 첫 업무 적용까지 걸린 시간
- 등록된 prompt 수
- 회고 제출률
- 발견된 blocker 수
- 비개발자 설치 성공 여부
```

## Pilot 종료 산출물

```text
- 설치 실패 목록
- profile 수정 목록
- rule 수정 목록
- 비개발자 UX 개선 목록
- 공용 계정 운영 이슈
- 전사 rollout 가능/불가 판단
```

## 완료 기준

- blocker가 0개이거나 workaround가 문서화된다.
- 비개발자도 설치 후 첫 업무를 수행한다.
- profile/rule 수정 사항이 반영된다.
- 전사 rollout 여부를 결정할 수 있다.
