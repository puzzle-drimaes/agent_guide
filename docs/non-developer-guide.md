# 비개발자 따라하기 — AI 업무 시작 가이드 (PM·기획·사업)

대상: 개발자가 아니어도 회사 AI 도구를 업무에 바로 쓰고 싶은 분.
터미널이나 코딩을 몰라도 됩니다. 위에서부터 순서대로 따라오세요.

## 0. 무엇을 하게 되나요

```text
- 회사 표준 방식으로 AI(Claude/Codex/Gemini/Kiro)를 업무에 씁니다.
- 검증된 프롬프트/스킬을 받아 쓰고, 잘 쓴 건 가볍게 공유합니다.
- 시작 방법은 두 가지:
  · 가벼운 방법(설치 없이): AI 대화창에 회사 프롬프트를 붙여 바로 사용 + 공유 폴더에서 받아 쓰기.
  · 정식 방법(프로젝트 설치): 자주 쓰는 폴더에 회사 설정을 한 번 설치.
- 처음엔 가벼운 방법만으로도 충분합니다.
```

## 1. 준비물

```text
- 회사에서 쓰는 AI 도구 접근(예: Claude, Kiro).
- 전사 공유 드라이브 "Drive" 접근(안 보이면 관리자에게 멤버 추가 요청).
- (정식 설치할 때만) 담당자가 주는 설치 묶음(zip).
```

## 2. 가장 쉬운 시작 (설치 없이)

```text
1. AI 도구 대화창을 연다.
2. "내 직무는 기획(또는 사업)이야. 회사 표준 방식으로 도와줘." 라고 말한다.
3. 공유 폴더 AI-Knowhow/prompts 에서 필요한 프롬프트 .md를 열어 내용을 복사해 붙여넣는다.
4. 빈칸(예: <고객명>)을 내 상황으로 바꿔 실행한다.
```

이게 제일 빠른 시작입니다. 마음에 들면 그다음에 정식 설치를 해도 됩니다.

## 3. 정식 설치 (원할 때, 한 번만)

설치는 명령을 외울 필요 없이 **"AI에게 시키는" 방식**입니다.

```text
1. 담당자가 준 설치 묶음(zip)을 풀어 그 폴더를 연다.
2. AI 대화창에 그 안의 SETUP_WIZARD.md 내용을 붙여넣고
   "이걸 읽고 내 설치를 도와줘" 라고 한다.
3. AI가 직무/도구/폴더를 물어본다 → 답한다(예: 기획 / Claude / 이 프로젝트 폴더).
4. AI가 먼저 "미리보기(dry-run)" 명령을 만들어준다 → 실행해 무엇이 생기는지 확인한다.
5. 괜찮으면 AI가 주는 설치(apply) 명령을 실행한다.
6. 명령을 어디에 넣는지 모르면 AI에게 "이 명령을 어디에 어떻게 입력해?" 라고 그대로 물어본다.
```

직무별 추천 profile:

```text
- 기획 / PM       → product
- 사업 / 영업 / CS → business
```

막히면 SETUP_WIZARD.md의 "설치가 안 될 때"를 AI에게 함께 보여주세요.

## 4. 공유 폴더에서 받아 쓰기 / 올리기 (브라우저면 충분)

받아 쓰기:

```text
1. 아래 폴더 링크를 직접 연다 (이름으로 검색하면 결과가 많으니 링크로 바로 가기).
     skills:  https://drive.google.com/drive/folders/1PZx9YNdpPZ2I2LX0zYC1k-XerQ0nEMys
     prompts: https://drive.google.com/drive/folders/1Cv104mmGpw9fqNQfuAttr_LfQl8Hxqee
2. 필요한 .md를 선택해 내려받는다.
3. AI 대화창에 내용을 붙여넣고 내 상황에 맞게 쓴다.
   (개발 프로젝트가 있으면 AI-Knowhow/ 에 넣고 AI에게 "배치해줘"라고 요청 —
    자세히는 agent-deploy/docs/SHARED_FOLDER_GUIDE.md)
```

올리기(공유):

```text
1. 내가 잘 쓴 프롬프트가 있으면 prompts/ 에 .md로 올린다(원하면).
2. 올리기 전, 고객 이름·연락처 같은 민감정보는 지우거나 <고객명>으로 바꾼다.
3. 양식은 자유. 부담 없이 올린다.
```

## 5. AI가 공유 폴더를 직접 읽게 하기 (선택)

```text
- 4번은 연동 없이 됩니다. AI가 폴더를 "직접" 읽게 하면 더 편한데, 이때만 한 번 연동합니다.
- 방법: AI 도구 설정 → Connectors/통합 → Google Drive → 연결(구글 로그인).
- 자세히: agent-deploy/docs/SHARED_FOLDER_GUIDE.md 의 "Google Drive 연동".
```

## 6. 첫 업무 예시

기획 / PM:

```text
- "이 아이디어를 PRD 초안으로 정리해줘."
- "고객 요청을 사용자 스토리와 완료 조건(acceptance criteria)으로 나눠줘."
```

사업 / 영업 / CS:

```text
- "이 고객 문의에 정중하고 정확한 답변 초안을 써줘."
- "이 회의 내용을 결정 / 할 일 / 리스크로 정리해줘."
- "이 제안서 핵심 메시지를 3개로 줄여줘."
```

## 7. 꼭 지킬 것 (딱 두 가지)

```text
1. 민감정보 입력 금지: 고객 개인정보, 비밀번호/토큰, 미공개 계약·재무 자료.
   필요하면 <고객명>처럼 가린다.
2. 외부 자료를 쓰면 출처를 남긴다.
```

그 외 기록 / 회고 / 등록은 강제가 아닙니다. 공유하고 싶을 때만 하면 됩니다.

## 8. 막히면

```text
- 설치 오류: SETUP_WIZARD.md "설치가 안 될 때"를 AI에게 보여주고,
  그래도 안 되면 AI에게 "이 내용을 피드백 .md로 정리해서 Google Drive AI-Knowhow/feedbacks에 올릴 수 있게 해줘"라고 요청한다.
- Drive 커넥터가 연결돼 있으면 AI가 직접 업로드를 도와주고, 없으면 업로드할 .md 본문을 만들어 준다.
- 그 외 질문/노하우도 원하면 AI가 feedbacks 또는 prompts/skills용 .md로 정리해 공유 폴더 업로드를 도와준다.
```
